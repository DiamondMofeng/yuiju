# World 侧主动发送消息技术方案

## 目标

- `chooseActionAgent` 在选择 Action 时同时判断是否想分享生活事件，并给出分享理由。
- Action 完成后，world 侧异步结合目标群上下文判断此刻是否发送，并生成最终群消息。
- 主动分享不阻塞 Action 生命周期，不影响下一次 tick。
- message 侧只提供内部 HTTP 能力：获取群聊上下文、获取表情包说明、发送并记录群消息。
- 第一版只做固定目标群，不做私聊、多群选择、重试、落库、Redis 节流。

## 主流程

```text
chooseActionAgent
  -> 选择 Action
  -> 判断是否想分享这次生活事件
  -> 输出 proactiveShareIntent

recoverRunningAction
  -> completionEvent 完成结算
  -> 更新 behavior episode
  -> scheduleActionCompletionProactiveShare
  -> clearRunningAction
  -> return eventDescription

主动分享异步任务
  -> 没有 proactiveShareIntent：结束
  -> 读取 message.proactive.groupTargetId
  -> GET /internal/stickers
  -> GET /internal/groups/:groupId/context
  -> world LLM 结合群聊上下文判断是否发送并生成消息
  -> shouldSend=false：记录日志后结束
  -> shouldSend=true：POST /internal/groups/:groupId/messages
```

`scheduleActionCompletionProactiveShare` 只调度异步任务，不 `await` 主动分享执行结果。

## 分享意图

`chooseActionAgent` 输出 Action 决策时，同时输出可选分享意图：

```ts
interface ActionAgentDecision {
  action: ActionId;
  reason: string;
  durationMinute?: number;
  planChanges?: AgentPlanChange[];
  proactiveShareIntent?: {
    shouldShare: boolean;
    reason: string;
  };
}
```

规则：

- `shouldShare=true` 表示悠酱主观上想把这次生活事件分享出去。
- `reason` 说明为什么想分享，用于完成后生成消息。
- 只有 `ActionMetadata.proactiveShare.enabled=true` 的 Action 才允许产生分享意图。
- 不在 Action 完成后再调用 LLM 判断“这件事是否值得分享生活”。

## Action 分享配置

扩展 `ActionMetadata`：

```ts
abstract class ActionMetadata {
  // ...
  proactiveShare?: {
    enabled: boolean;
  };
}
```

规则：

- 不配置时默认不可分享。
- `proactiveShare.enabled=true` 只表示该 Action 允许产生分享意图。
- 是否真的想分享，由 `chooseActionAgent` 根据当前状态和行动理由决定。
- 是否适合发到群里，由 Action 完成后的发送决策 LLM 根据群聊上下文决定。

第一版配置：

```text
Study_At_School proactiveShare.enabled=true
Eat_Breakfast proactiveShare.enabled=true
Eat_Lunch proactiveShare.enabled=true
Eat_Dinner proactiveShare.enabled=true
Eat_Item proactiveShare.enabled=true
Buy_Item_At_Shop proactiveShare.enabled=true
Order_Coffee proactiveShare.enabled=true
Drink_Coffee proactiveShare.enabled=true
Work_At_Cafe proactiveShare.enabled=true

Stay_At_Home proactiveShare.enabled=true

```

普通移动、发呆、短暂停留等低信息量 Action 不配置。

## 数据输入

主动分享任务需要输入：

```ts
interface ActionCompletionProactiveShareInput {
  action: ActionId;
  shareIntent: {
    reason: string;
  };
  behaviorEpisodeId: string;
  eventDescription?: string;
  completionContext?: Record<string, unknown>;
  characterStateSnapshot: Record<string, unknown>;
  worldStateSnapshot: Record<string, unknown>;
}
```

说明：

- `shareIntent` 来自 `chooseActionAgent`。
- `eventDescription` 和 `completionContext` 来自 `completionEvent`。
- `characterStateSnapshot` / `worldStateSnapshot` 只作为 LLM 上下文，不作为新真相源。
- `behaviorEpisodeId` 只用于日志追溯，不落库主动分享结果。

## message 内部 API

world 侧调用 message 侧已实现的内部 HTTP 能力：

```text
GET /internal/stickers
GET /internal/groups/:groupId/context?limit=20
POST /internal/groups/:groupId/messages
```

接口地址来自：

```ts
message.internalApi.host
message.internalApi.port
```

目标群来自：

```ts
message.proactive.groupTargetId
```

## 发送决策 LLM 输出

Action 完成后的 LLM 只判断当前群聊上下文是否适合发送，并生成最终消息；不再判断“是否想分享生活事件”。

```ts
interface ProactiveGroupMessageDecision {
  shouldSend: boolean;
  reason: string;
  message: string;
}
```

约束：

- `shouldSend=false` 时不发送。
- `message` 是最终发到群里的正文。
- `reason` 说明为什么当前适合或不适合发送。
- 允许使用 message 侧提供的 `[[sticker:key]]` 表情包标记。
- 不提 Action、completionEvent、触发记录、内部接口等实现概念。

## Prompt 输入

Prompt 包含：

- `chooseActionAgent` 产生的分享意图和理由。
- Action 完成事实。
- completionContext。
- 当前角色状态摘要。
- 当前世界时间、天气等摘要。
- 目标群最近消息上下文。
- message 侧表情包提示词。
- 悠酱角色设定与群聊回复规则。

LLM 负责判断：

- 当前群聊上下文是否适合插入这条生活分享。
- 应该怎么自然地说。

## 错误处理

- 主动分享异步任务内部 catch 错误并记录日志。
- 获取 message 内部 API 失败：记录日志后结束。
- LLM 失败：记录日志后结束。
- 发送失败：记录日志后结束。
- 任何失败都不回滚 Action 完成结果，不影响 `clearRunningAction`。

## 第一版不做

- 主动分享结果落库。
- Redis 去重与节流。
- 失败自动重试。
- 延迟发送。
- 私聊主动消息。
- 多群目标选择。
- LLM 选择发送渠道。
- 由 message 侧判断是否分享。
