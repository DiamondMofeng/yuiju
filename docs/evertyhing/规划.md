## 群 bot 互动文字小游戏

新建一个群 bot，用来承载文字小游戏前端，可以与悠酱一起互动。

### 悠酱电波对齐

群友猜测悠酱在某个轻量情境下会做出的选择。玩法重点是“理解悠酱的想法”，适合用投票快速完成一局。

基本流程：

1. bot 发出一个情境题和 2-4 个选项。
2. 群友在限定时间内投票，例如回复 `A`、`B`、`C`。
3. 时间结束后公布悠酱的真实选择。
4. 猜中的群友获胜，并生成一段简短反馈。

示例：

```text
【悠酱电波对齐】
下雨天，悠酱最想做什么？

A. 睡觉
B. 看番
C. 出门买奶茶

30 秒内投票：A / B / C
```

结算示例：

```text
悠酱选择：B. 看番

猜中的人：小明、小红
悠酱：下雨天窝在家里看番才是正解，出门是不可能出门的。
```

第一版可以先使用固定题库和固定答案，不需要接入 LLM。后续如果需要增加变化，可以让 LLM 生成情境、选项和结算文本，但不影响核心规则。

### 反向猜词

群友看到目标词后，用文本描述帮助悠酱猜出答案。玩法重点是让悠酱作为参与者，而不是只做裁判。

基本流程：

1. bot 公布一局开始，并把目标词发给参与群友。
2. 群友不能直接说出目标词，只能用一句话描述。
3. 悠酱根据群友描述进行猜测。
4. 悠酱在限定轮数或限定时间内猜中，则群友和悠酱共同胜利；否则本局失败。

示例：

```text
【反向猜词】
目标词：奶茶

请不要直接说出目标词，用一句话让悠酱猜到它。
```

群友描述：

```text
一种能让下午重新活过来的甜甜饮料，通常有吸管。
```

悠酱猜测：

```text
悠酱：是奶茶吗？
```

结算示例：

```text
猜对了！
悠酱：这个描述太明显了，下午活命装置只能是奶茶。
```

第一版可以先用固定词库，并限制每局最多猜 3 次。为了降低实现复杂度，禁词校验可以先只检查是否直接包含目标词；更复杂的同义词、谐音和拆字规避暂不处理。
